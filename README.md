# Case Studies

Write-ups of work I can't open-source. The code is private (either under NDA or in active development as a product) but the engineering and the thinking are worth showing.

Each case study follows the same shape: **the problem → the approach → the architecture → the outcome and impact → production considerations → why the code stays private.** They're written to show the part that matters for delivery and strategy roles: translating a messy real problem into a system, the decisions and trade-offs behind it, and what it would take to run in production. I'm happy to go much deeper in conversation, including a live walk-through.
